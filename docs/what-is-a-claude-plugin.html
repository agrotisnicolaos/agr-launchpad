Field Guide · Claude Code

What is a
Claude plugin?

A beginner-first tour of the one-folder packages that teach Claude Code new tricks — no prior knowledge needed.

01Why should you care?

You're probably using plugins already without thinking about them. In a typical session:

  • A memory plugin remembers what you worked on last night and briefs Claude this morning
  • A workflow plugin nudges Claude to follow disciplined habits like test-driven development
  • A design plugin gives Claude visual taste when you ask for a web page
A Claude plugin is one installable folder that teaches Claude Code new tricks — knowledge, helpers, automatic reactions, and connections to outside tools — all behind a single on/off switch.

02The building blocks

Out of the box, Claude Code is a capable generalist. A plugin upgrades it — and one plugin can bundle up to five kinds of upgrades:

📇Skills

Written instructions Claude reads only when relevant.

Like recipe cards in a kitchen drawer — out of the way until the right dish is ordered.

👷Agents

Pre-configured specialist coworkers Claude can hand sub-tasks to.

Like a contractor calling in an electrician instead of doing the wiring himself.

⚡Commands

Shortcuts you type as /something to trigger a whole workflow.

Like speed-dial — one button, a whole routine runs.

🔔Hooks

Automatic reactions: “when X happens, always do Y.”

Like a doorbell — nobody has to remember to announce visitors.

📞MCP servers

Phone lines from Claude to outside tools — GitHub, your browser, your database.

Like giving your assistant the office phone directory.

🏷️The manifest

A small label file (plugin.json) saying “here's my name, version, and what's inside.”

Like the ingredients label on a box. 
PLUGIN (one folder)
├── plugin.json   → the label: name, version, contents
├── skills/       → recipe cards, read on demand
├── agents/       → specialist coworkers
├── commands/     → /speed-dial shortcuts
├── hooks/        → doorbells ("when X, do Y")
└── mcp servers/  → phone lines to outside tools

A plugin doesn't need all five — many ship just one skill. The point is the packaging: one folder, one install, one uninstall. Plugins live in marketplaces — catalogs (usually just a GitHub repo) you browse with /plugin. Like an app store, except anyone can run one.

03Putting it together

author writes folder → publishes to a marketplace → you run /plugin install → Claude reads the manifest → everything switches on

Walking through the memory plugin from the hook:

  1. Someone wrote a “memory for Claude” plugin and published it to a marketplace
  2. You installed it once with /plugin
  3. Its hook rang last night when your session ended → it saved a summary
  4. This morning, another hook injected those memories before Claude read your message
  5. Its skill waits in the drawer in case you ask “did we solve this before?”
  6. You never configured any of this individually — installing the plugin wired it all up
One-step setupA whole workflow — hooks, skills, tools — installs and uninstalls as a unit, instead of hand-editing config files.
ShareabilityYour whole team gets identical Claude behavior by installing the same plugin.

04Successful examples in the wild

  • obra/superpowers

    The most popular community plugin — enforces disciplined workflows like test-driven development.

  • anthropics/skills

    Anthropic's own official collection: document handling, spreadsheets, PDFs.

  • Trail of Bits

    A respected security firm packaged 40+ of their audit practices as plugins.

  • agr-launchpad

    A curated starter kit bundling plugins and skills — proof that a plugin collection can be a product in itself.

05Best practices & common mistakes

✅ Do❌ Don't
Install from sources you trust — plugins run code and read your files Treat marketplaces like a vetted app store; anyone can publish
Write skill descriptions that say exactly when to trigger Write vague descriptions — Claude won't know when to use the skill
Keep one plugin = one coherent purpose Cram unrelated tools into a mega-plugin that's hard to audit
Disable plugins you stopped using Let a dozen idle plugins inflate every session's startup context
