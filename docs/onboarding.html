1What this is

agr-launchpad is a starter repository for Claude Code — Anthropic's AI coding assistant that lives in your terminal and editor. Instead of starting every project from an empty folder, you start from here: a curated, battle-tested foundation that makes Claude noticeably more capable and disciplined.

Two layers:

Base

The launchpad

Universal, project-agnostic. Great skills, the best general plugins, helpful assistants, and live tools. This is everything you need to start anything.

Packs

Project add-ons

When you start a specific kind of project (ML, an iOS app, a journal…), you install a pack that layers on the right extra capability. The base stays clean.

You don't need to be a programmer to use this. If you can clone a folder and click a few “Yes” buttons, you can get set up. The rest is just talking to Claude in plain English.

3Setup — about 5 clicks

  1. Clone the repo and open the folder in Claude Code (or in VS Code with the Claude Code extension). 
    git clone <your-repo-url> agr-launchpad
cd agr-launchpad
  2. Trust the folder. Claude Code asks “Do you trust the files in this folder?” → click Yes, I trust. This unlocks the bundled setup.
  3. Approve the plugins. Claude offers to add two marketplaces and install six plugins. Approve them. (Details in section 4.)
  4. Approve the MCP servers. Claude asks about context7, github, and jupyter. Approve them. (Details in section 5.)
  5. Add secrets (optional). Copy the example env file and fill in a GitHub token only if you want the GitHub integration: 
    cp .env.example .env
    Prefer a guided walkthrough? Run make setup.
  6. Restart Claude Code so everything loads. Done — start describing what you want to build.

Why the approvals? Claude Code never installs plugins or connects tools behind your back — you approve each one once. That's a safety feature, not a hurdle.

4The plugins

Why aren't these already installed? Claude Code plugins live in your Claude setup (your global ~/.claude), not inside a project folder — so no repository can physically ship them pre-installed. What this repo does is pre-configure them (in .claude/settings.json) so Claude Code installs them for you with a couple of clicks. You never hunt them down by hand.

Two ways to install them

A — Automatic (recommended). When you trust the folder, Claude Code detects the pre-configured plugins and offers to add two marketplaces and install all six. Click Approve on each prompt. That's it.

B — One command. Prefer the terminal, or the prompts didn't appear? Run:

make install-plugins

This uses the Claude Code CLI to add the marketplaces and install every plugin in one go. Restart Claude Code afterward.

Verify any time: type /plugin in Claude Code (or run claude plugin list in a terminal) — all six should be listed and enabled.

What each one gives you

superpowers

Disciplined workflows: brainstorming before coding, writing plans, test-driven development, systematic debugging, verifying before claiming done.

skill-creator

Build your own custom skills when you find yourself repeating a task. Claude scaffolds and tests them for you.

frontend-design

Anthropic's skill for distinctive, production-grade user interfaces — avoids the generic “AI look”.

code-review

/code-review checks your changes for bugs and cleanups. /code-review ultra runs a deep multi-agent review in the cloud.

context-mode

Keeps large command output out of the conversation so Claude stays fast and focused on long tasks.

claude-mem

Persistent memory — Claude remembers what you worked on across sessions, so you don't re-explain your project every time.

5The MCP servers

MCP servers give Claude live connections to the outside world. Three are already declared in this repo's .mcp.json, so you don't add them by hand — you just approve them once and (for two of them) do one small setup step.

Step 1 — Approve them

When you trust the folder, Claude Code asks whether to enable the project's MCP servers. Click Approve. To check or re-approve later, run claude mcp list in a terminal, or type /mcp inside Claude Code.

Step 2 — Per-server setup

ServerWhat it doesWhat you do
context7Pulls the current docs for any library or framework, so Claude doesn't rely on stale training data. Nothing. Works as soon as it's approved (it runs via npx, which Node provides).
githubReads and manages your GitHub repos, issues, and pull requests. Add a token — see just below. Skip if you don't want GitHub features yet.
jupyterLive notebook kernel: Claude can run and edit .ipynb cells. Run make jupyter in a terminal and leave it open. (Needs uv.)

Giving GitHub a token (only for the github server)

  1. Copy the env template once: 
    cp .env.example .env
  2. Create a token at github.com/settings/tokens → Generate new token (a fine-grained token with read access to your repos is plenty).
  3. Open .env and paste it after the equals sign: 
    GITHUB_PAT=github_pat_xxxxxxxxxxxx
  4. Restart Claude Code. Verify with claude mcp list — github should show as connected.

Your token stays private. .env is git-ignored — it is never committed or shared. Leave GITHUB_PAT blank to skip GitHub entirely; the other two servers still work.

7Packs — add capability for a project

The base is deliberately lean. A pack is how you add capability for a specific kind of project. A pack isn't limited to one type of thing — it can bundle any mix of:

Skills

New /slash commands for that domain.

Agents

Specialist subagents Claude can dispatch.

MCP servers

Extra live tools/data connections.

Plugins

Whole plugins, pulled in for that project type.

Rules

A rulebook fragment added to CLAUDE.md.

Templates & tools

Any other assets the project needs.

In short: a pack can add plugins or any other asset, not just skills.

Two ways packs reach you

Local packs bundled in this repo's packs/ folder — manage them with:

make list-packs           # see what's available
make new-pack name=ml     # create a new pack from the template
make use-pack name=ml     # install a pack into your setup
make unuse-pack name=ml   # remove it again

Installing copies the pack's skills and agents into place, adds any MCP servers it needs, and appends its rules to CLAUDE.md — all reversible. Only the _template ships filled in; it's a worked example you copy to build your own (see packs/README.md).

Published packs come from agr·hub — a marketplace this repo already knows about (it's pre-registered in .claude/settings.json). So when a new pack is ready, you install it with a single command, no setup:

/plugin install ml@agr-hub

That's how new packs are shared over time: you'll be told a pack is ready, and one /plugin install adds it — skills, agents, tools and all. Browse what's available at agrotisnicolaos.github.io/agr-hub.

8gsd (installed separately — optional)

gsd (“get shit done”) is a powerful planning toolkit — it breaks big goals into phases, plans, and verified steps. It installs globally into your Claude setup (~/.claude) rather than from this repo, so once installed it works across all your projects. That's also why it isn't in make install-plugins — it's its own installer.

Install

Run this once in a terminal (needs Node.js):

npx get-shit-done-cc@latest

Follow its prompts, then restart Claude Code. Its /gsd-* commands now appear in every project. Update later with the same command, or /gsd-update inside Claude. Everything else in this kit works without gsd — install it only if you want milestone-scale planning.

9markitdown (turn any file into Markdown — optional)

The markitdown VS Code extension converts PDFs, Word/Excel/PowerPoint, HTML, and images into clean Markdown — perfect for feeding documents to Claude. It's a VS Code extension, so it installs through VS Code, not through Claude Code.

Install

  1. Open this folder in VS Code. A toast appears: “This workspace has extension recommendations.” Click Install All — that's the fastest path (it's recommended via .vscode/extensions.json).
  2. No toast? Open the Extensions panel (⌘⇧X / Ctrl+Shift+X), search MarkItDown (publisher BioInfo), and click Install.
  3. Use it: right-click a supported file → Convert to Markdown. (If it asks for the underlying tool, the extension's README covers the one-time markitdown dependency.)

10Your first project

A natural flow once you're set up:

  1. Tell Claude what you want to build. For anything non-trivial, it'll brainstorm with you first.
  2. Want to pressure-test your idea? Type /grill-me and defend it.
  3. Claude writes a plan, then implements it with tests (superpowers keeps it honest).
  4. Before you ship, run /code-review — or /code-review ultra for the deep pass.
  5. Need a domain boost partway through? make use-pack name=<pack> and keep going.

You mostly just talk. The skills and plugins make Claude ask the right questions, plan before coding, and check its own work — so you get further with less back-and-forth.

11Troubleshooting & FAQ

I didn't get the plugin install prompts

Make sure you trusted the folder, then restart Claude Code. Or skip the prompts entirely and run make install-plugins in a terminal — it installs all six via the Claude Code CLI. Confirm with /plugin or claude plugin list.

The GitHub tools aren't working

Check that .env exists and GITHUB_PAT is filled in, then restart.

Jupyter says it can't connect

Run make jupyter in a terminal and leave it running; it starts the kernel the jupyter MCP talks to.

Is my data safe to share this repo?

Yes — no secrets are committed. .env is git-ignored. Double-check it's not staged before pushing anywhere public.

Can I add my own skills?

Absolutely — drop a folder in .claude/skills/, or ask Claude to use skill-creator. For project-specific ones, put them in a pack.
