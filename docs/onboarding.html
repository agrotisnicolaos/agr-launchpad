agr-launchpad — a friendly, complete guide

No coding experience needed. This guide shows you exactly what to install, where to type each thing, and what to say — so you can set up the launchpad correctly and start building with Claude today. Read top to bottom; it takes about 15 minutes.

1What this is, and why it's worth it

Claude Code is an AI assistant from Anthropic that helps you build things — websites, small apps, scripts, charts, documents — by chatting with it in plain English. agr-launchpad is a ready-made starting folder for Claude Code. Open it, and Claude instantly becomes more capable, careful, and helpful.

Think of it like a well-stocked workshop. An empty room means you fetch every tool yourself. The launchpad hands you the workshop already set up — the right tools, arranged so they work together — so you can just start making things.

What you get

A smarter assistant

Claude plans before it builds, writes real tests, reviews its own work, and remembers your project between sessions.

What you give

A few clicks

Clone a folder, click “approve” a few times. No coding, no configuration files to edit by hand.

The promise

Freedom + a safety net

Build however you like. The launchpad quietly assists you the whole way — and you can customize it as much as you want, later.

You do not need to be a programmer. If you can install an app and click a few buttons, you can do everything in this guide. We explain every step.

2How to read this guide — the 2 places you type

This is the one idea that makes everything click. There are two different places you'll type things, and it matters which is which. Throughout this guide, every instruction is tagged so you always know where it goes:

🖥 Terminal

The Terminal is a plain text window where you run setup commands (like installing things). You type a command, press Enter, and it runs. You'll only need it a handful of times.

an example command you'd paste into the Terminal
💬 Claude

This is the chat box inside Claude Code — where you talk to the AI in normal sentences. This is where you spend almost all your time once you're set up.

You say to Claude

“Build me a simple webpage with my name and a short bio.”

Rule of thumb: 🖥 Terminal = setup commands (rare). 💬 Claude = everything you actually want to make (all the time). When in doubt, look at the tag above the box.

3What you need (and exactly how to get it)

Two free programs, installed once. Take them one at a time.

3.1 — Claude Code

  1. Download it

    Go to claude.com/claude-code and install the app for your computer (Mac or Windows). It's also available as a VS Code extension.

  2. Sign in

    Open it and sign in with your Anthropic account (create one if you don't have it).

3.2 — Node.js

Some of the launchpad's tools run on Node.js — a small free engine. You don't use it directly; it just needs to be present.

  1. Download the “LTS” version

    Go to nodejs.org and click the big LTS button. Open the downloaded file and click through the installer (all defaults are fine).

  2. Check it worked

    Open your Terminal (next box shows how) and type: 🖥 Terminal 
    node -v
    If you see a version number like v22.x.x, you're good.

How do I open the Terminal?

MacWindows
Press ⌘ Space, type Terminal, press Enter. Press the ⊞ Windows key, type Terminal (or PowerShell), press Enter.

Using the Terminal, in one sentence: click into the window, type (or paste) the command exactly, and press Enter. To paste: ⌘V on Mac, Ctrl+V (or right-click) on Windows.

Optional extras (only if you'll use them): uv for live notebooks, and git if you want to clone instead of download. Both are covered where they come up.

4Get the launchpad onto your computer

Two ways. Pick one. If you're not sure, choose Option A.

Option A — Download (easiest, no Terminal)

  1. Open the project page

    Go to github.com/agrotisnicolaos/agr-launchpad.

  2. Download it

    Click the green <> Code button, then Download ZIP. Find the ZIP in your Downloads and double-click to unzip it.

  3. Move it somewhere you'll find it

    Drag the unzipped agr-launchpad folder to your Desktop or Documents.

Option B — Clone (one Terminal command)

🖥 Terminal 
git clone https://github.com/agrotisnicolaos/agr-launchpad.git

This creates an agr-launchpad folder wherever your Terminal is currently pointed (usually your home folder).

Then: open the folder in Claude Code

  1. Open Claude Code (or VS Code)

  2. Open the folder

    Use File → Open Folder… and choose the agr-launchpad folder you just got.

5Set it up (about five clicks)

When you open the folder, Claude Code does most of the work — you just approve.

  1. Trust the folder

    A box asks “Do you trust the files in this folder?” → click Yes, I trust. This unlocks the pre-configured setup.

  2. Approve the plugins

    Claude offers to add two “marketplaces” and install six plugins. Click Approve on each. (What these are: section 6.)

  3. Or install them with one command

    Prefer the Terminal, or didn't see the prompts? In the launchpad folder, run: 🖥 Terminal 
    make install-plugins

  4. Approve the live tools

    Claude asks about four “MCP servers” (context7, github, jupyter, chrome-devtools). Click Approve for the ones you want — chrome-devtools is optional (it lets Claude look at web pages it builds for you; decline it if unsure). (GitHub needs a token — optional, see below.)

  5. Restart Claude Code

    Close and reopen it so everything loads. Done.

Check it worked: in the Claude chat, type /plugin — you should see all six plugins listed and enabled.

Optional: connect GitHub

Only if you want Claude to work with your GitHub repositories. You give it a private “token”.

  1. Make a copy of the settings file

    🖥 Terminal 
    cp .env.example .env

  2. Create a token

    Go to github.com/settings/tokens → Generate new token (a fine-grained token with read access to your repos is plenty). Copy it.

  3. Paste it in

    Open the new .env file and put your token after the equals sign: 
    GITHUB_PAT=github_pat_xxxxxxxxxxxx
    Save the file.

  4. Load it when you start Claude Code

    Claude Code reads the token from your Terminal session, not from the file directly. Start Claude Code from the Terminal like this: 🖥 Terminal 
    set -a; source .env; set +a; claude
    (Tip: install direnv once and this happens automatically.)

Your token stays private. The .env file is never uploaded or shared. Leave it blank to skip GitHub entirely — everything else still works.

Ready-made skills & agents — already in the folder

No installing needed. Skills are invoked with a slash, like /grill-me, or picked up automatically when relevant. Highlights: /grill-me (stress-tests your plan with questions), coding-standards, error-handling, secure-coding (safety checks while writing sensitive code), api-design + backend-patterns + frontend-patterns (solid foundations for both halves of an app), html (polished one-file web pages), webapp-testing (Claude screenshots its own work to verify it), code-tour. Helper “agents” quietly plan and explore your project when needed.

7Your first project — with example prompts

Here's the magic part: you just describe what you want, in the 💬 Claude chat. Below are real prompts you can copy. Claude will usually ask a couple of questions, make a short plan, then build.

Make a simple website

You say to Claude

“Build me a clean, modern one-page website with my name (Alex Rivera), a short bio about me being a photographer, and buttons linking to my Instagram and email. Then show it to me.”

Organize or tidy files

You say to Claude

“I have a folder of photos with messy names. Help me rename them all to the date each photo was taken, like 2026-06-09-01.jpg.”

Turn data into a chart

You say to Claude

“I have a spreadsheet of my monthly expenses called budget.csv. Make me a simple chart showing which categories I spend the most on.”

Learn while you build

You say to Claude

“I'm brand new to coding. Build a tiny number-guessing game with me and explain each step in plain English as we go.”

Be specific, then relax. The more detail you give (names, colors, what it's for), the better the result. After that, let Claude drive — it'll plan, build, and check its work for you.

A natural rhythm

  1. Describe what you want. For anything non-trivial, Claude will brainstorm with you first.
  2. Want to pressure-test your idea before building? Type /grill-me and answer its questions.
  3. Claude makes a plan, then builds it (writing tests as it goes).
  4. Before you rely on it, type /code-review for a once-over.

Packs — capability for a specific kind of project

When you start, say, a data project or an app, you install a pack: a ready bundle of skills, agents, and tools for that domain. Two ways they reach you:

Packs bundled in the folder — manage them in the Terminal:

🖥 Terminal 
make list-packs            # see what's available
make use-pack name=ml      # add a pack
make unuse-pack name=ml    # remove it

Published packs — install from a hub with one line in the 💬 Claude chat:

💬 Claude 
/plugin install ml@agr-hub

New packs are shared over time through agr·hub (agrotisnicolaos.github.io/agr-hub) — and can come from other hubs and marketplaces too. You'll be told when one is ready; a single /plugin install adds it, skills and tools and all.

10Optional power-ups

gsd — for bigger, multi-step plans

A planning toolkit that breaks large goals into phases and verified steps. It installs separately (it works across all your projects). Run once in the Terminal:

🖥 Terminal 
npx get-shit-done-cc@latest

Follow its prompts, then restart Claude Code. Its /gsd-* commands then appear everywhere. Optional — skip it unless you want milestone-scale planning.

markitdown — turn PDFs & docs into text Claude can read

A VS Code extension that converts PDFs, Word/Excel/PowerPoint, and images into clean Markdown. When you open the folder in VS Code, a box suggests it — click Install All. Or search MarkItDown (by BioInfo) in the Extensions panel.

11Troubleshooting & FAQ

I didn't see the “install plugins” prompts.

Make sure you clicked Yes, I trust on the folder, then restart Claude Code. Or just run make install-plugins in the 🖥 Terminal. Confirm with /plugin.

A command “isn't found” or gives an error in the Terminal.

Two common causes: (1) you're not inside the launchpad folder — see the next question; or (2) Node.js isn't installed — re-check section 3 with node -v.

How do I make sure the Terminal is “in” the launchpad folder?

Type cd (with a space), then drag the agr-launchpad folder onto the Terminal window and press Enter. That moves you into it. Tip: opening the Terminal inside Claude Code/VS Code usually starts you there already.

Do I have to use the Terminal at all?

Barely. You can set up with clicks (Download ZIP + approve prompts) and do everything else by chatting with Claude. The Terminal is only for a few optional commands.

The GitHub features aren't working.

Check that you created the .env file, pasted your token into GITHUB_PAT, and started Claude Code with the token loaded (set -a; source .env; set +a; claude — see section 5). No token? That's fine — GitHub is optional.

Jupyter / notebooks won't connect.

Run make jupyter in the 🖥 Terminal and leave that window open — it powers the live notebooks. (It needs uv installed.)

Is it safe to share this folder?

Yes — no secrets are included, and your .env token is never uploaded. Just don't share your .env file itself.

Can I break something?

Not really. It's your own copy. If something gets messy, delete the folder and download a fresh one. And Claude tracks its changes, so you can always ask it to undo.

What if I want more for a specific project?

Add a pack (section 9). The base stays simple; you bolt on exactly the extra power you need, when you need it.
